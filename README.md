## Your challenge

Your neighbor, the local artist, is back with good news: they have launched an online store to sell tickets for their performances. The problem is that although the site is getting many visits and purchases, nobody can answer basic questions such as:

- How much money are we making?
- How many tickets have been sold and how many are left?
- How well does the site convert (purchases / visits)?
- Which promotions work and which are cannibalizing the price?
- Who has bought the most tickets?

The artist contacts you because they need a clear report (a dashboard) that lets them oversee the business without getting lost in data. And you, although you are just getting started, want to deliver a solid, professional proposal—so your mission is to design a dashboard that shows them what matters most for making quick decisions.

---

### What makes a dashboard “good”?

Organize the dashboard into broad blocks:

**1️⃣ Top block:** Main outcome indicators (KPIs):

- **Volume:** sales / sign-ups / active users
- **Revenue:** revenue, MRR, average ticket
- **Retention:** churn, persistence, completion
- **Satisfaction:** NPS / CSAT
- **Efficiency:** cost per outcome, margin, lead times

**2️⃣ Middle block:** “Drivers” (factors that explain the outcome):

- **Conversion** by stage (funnel)
- **Quality** (qualified lead ratio, attendance rate, pass rate)
- **Activity** (touches, sessions, deliveries, attendance)
- **Capacity** (occupancy, available slots, mentor/student ratio)

**3️⃣ Bottom block:** Operational details (tables/lists, alerts):

- Backlog, response times, SLA
- Alerts: sharp drops, spikes, anomalies
- Lists and tables with filters: “top issues”, “top opportunities”

## 💻 What you need to do

Create a dashboard that allows identifying at least **three KPIs**, **three drivers**, and operational details (tables/lists), as mentioned above. The dashboard must display correctly on any device (mobile, tablets, and desktops).

- [ ] First, identify the visual elements (components) you need. For example:
  - Does it need a navbar/header or sidebar?
  - How many sections does the main content split into?
  - What types of charts can you use for each indicator?
- [ ] Start with the mobile layout, which is the most constrained format, but always think about how those elements will translate to desktop.
- [ ] For charts you can use [Chart CSS](https://chartscss.org/), a CSS-only library (data can be made up; what matters is visual hierarchy and organization).

⚠️ **IMPORTANT:** In this project we only use **HTML and Tailwind CSS**. Make sure your AI Copilot **does not include more advanced technologies** (e.g. React). State this from the beginning of the work.
